AI Chief-of-Staff Agent

An autonomous execution system that ensures nothing critical slips through the cracks.

// Impacts (Metrics)

  • • Reduced decision latency from hours to <10 minutes
  • • Ensure 100% address segment or escalation within SLA (15 mins)
  • • Improved Signal to noise ratio by 85% using LLM classification
  • • Created full operational visibility across incident lifecycles

// Problem Statement

In fast-scaling teams, critical issues get buried in Slack, emails, and sheets. No ownership tracking, no acknowledgement system, no escalation logic if ignored, and high MTTR/missed SLAs mean existing systems notify problems—but don't ensure resolution.

// Solution Architecture

I built an AI Chief-of-Staff Agent—an autonomous loop that:

  • Monitors Slack, Gmail, and Sheets in real-time.
  • Classifies urgency using LLM (Gemini).
  • Tracks every issue via a state machine (Pending -> Acknowledged -> Escalated).
  • Forces accountability with human-in-the-loop acknowledgement.
  • Auto-escalates unresolved tasks via secondary channels.

// How It Works (System Design Workflow)

Scenario 1: Intelligent Filtering

The "Ingest Agent" monitors Slack messages in real-time. Instead of simple keyword matching, it leverages Google Gemini AI to understand context and sentiment. Only "Critical" incidents are assigned a unique 6-digit ID and logged into the "State Machine".

Scenario 2: Human-in-the-loop Acknowledgement

The "Accountability Layer" alerts are dispatched to Telegram. The system waits for a structured human response (Acknowledge Task). Upon receipt, it performs a surgical update to the database, transitioning the incident state from Pending to Acknowledged.

Scenario 3: Automatic Escalation

The "Fail Safe" background worker (Scheduler) audits the database every 15 minutes. It identifies "Ghost Incidents"—those that are still Pending past the SLA. It bypasses standard channels and triggers a Pushbullet high-priority alert to ensure immediate executive attention.

AI Feedback Intelligence System

Turning unstructured customer feedback into real-time product decisions.

// Impacts (Metrics)

  • • Reduced feedback processing time by 90% (15-20 hrs -> minutes)
  • • Decreased feedback-to-action cycle from 1 week to <15 minutes
  • • Reduced LLM operational costs by ~70% (DeepSeek optimization)
  • • Enabled real-time visibility into customer sentiment & product issues

// Problem Statement

Modern SaaS products receive thousands of feedback inputs daily across channels (support tickets, surveys, Slack, emails). However, feedback is unstructured and scattered, teams rely on manual tagging, there is no real-time visibility, and PMs are often forced to guess priorities, leading to misaligned roadmaps.

// System Pipeline

1. Data Ingestion: Raw feedback is captured in Airtable, which acts as the control data warehouse.

2. Logic Orchestration: A Make.com scenario is triggered by a "Blank Status" filter, ensuring only new data is processed to save computational costs.

3. AI Intelligence Layer: Data is sent to DeepSeek-V3 to perform a three-step cognitive task:

  • Sentiment Scoring: Classifies tone (Positive, Neutral, Negative).
  • Categorization: Labels feedback by product domain (UI/UX, Performance, Billing, etc.).
  • Abstractive Summarization: Generates a one-sentence "Executive Summary".

4. Data Bridging: Processed insights are synced back to Google Sheets to serve as a lightweight database engine.

5. Visualization: Looker Studio pulls the clean data to generate real-time, interactive heatmaps and sentiment trends.
